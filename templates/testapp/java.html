Java Exam Section

Exam Overview

The Java exam is designed to evaluate your understanding of Java programming fundamentals, object-oriented concepts, data structures, exception handling, and advanced Java features.

Exam Structure

  • Duration: 120 minutes
  • Total Questions: 60
  • Question Types: Multiple Choice, True/False, Code Analysis, Debug Code, and Programming Problems
  • Passing Score: 65%

Topics Covered

  1. Java Fundamentals - Syntax, Data types, Variables, Operators, Type casting
  2. Control Flow - If-else, Switch statements, Loops (for, while, do-while)
  3. Arrays - Single and Multi-dimensional arrays, Array operations
  4. Object-Oriented Programming - Classes, Objects, Constructors, this keyword
  5. Inheritance - extends keyword, super keyword, Method overriding
  6. Polymorphism - Method overloading, Runtime polymorphism
  7. Abstraction - Abstract classes, Interfaces, Abstract methods
  8. Encapsulation - Access modifiers, Getters and Setters
  9. Exception Handling - try-catch-finally, throw, throws, Custom exceptions
  10. Collections Framework - ArrayList, LinkedList, HashMap, HashSet, TreeSet
  11. String Handling - String class, StringBuilder, StringBuffer
  12. File I/O - FileReader, FileWriter, BufferedReader, BufferedWriter
  13. Multithreading - Thread class, Runnable interface, Synchronization
  14. Packages - Import statements, Creating packages, Access control

Question Distribution

  • Multiple Choice Questions: 30 questions (1 mark each)
  • True/False: 10 questions (0.5 marks each)
  • Code Analysis: 10 questions (2 marks each)
  • Debug Code: 5 questions (3 marks each)
  • Programming Problems: 5 questions (4 marks each)

Java Exam Specific Rules

  1. Java version used for evaluation: Java 8 or higher
  2. All code must follow proper Java naming conventions (camelCase for variables, PascalCase for classes)
  3. Include proper package declarations if required
  4. Use appropriate access modifiers for classes and methods
  5. Code must be properly indented and formatted
  6. Import statements should be included where necessary
  7. For programming questions, include main method for execution
  8. Compilation errors will result in zero marks for that question
  9. Use of deprecated methods will result in partial marks deduction
  10. Comments explaining logic are recommended

Sample Questions

Multiple Choice:

Q. Which of the following is not a Java feature?

a) Object-Oriented
b) Platform Independent
c) Pointer Arithmetic
d) Multithreaded

Code Analysis:

Q. What will be the output of the following code?

public class Test {
public static void main(String[] args) {
    String s1 = "Hello";
    String s2 = "Hello";
    String s3 = new String("Hello");
    System.out.println(s1 == s2);
    System.out.println(s1 == s3);
}
}

Debug Code:

Q. Find and fix the errors in the following code:

public class Calculator {
public int add(int a, int b) {
    return a + b;
}

public int add(int a, int b) {
    return a + b + 10;
}
}

Programming Problem:

Q. Write a Java program to implement a simple Bank Account class with deposit, withdraw, and check balance methods. Include proper exception handling for insufficient balance.

Important Java Concepts to Remember

  • Wrapper Classes: Integer, Double, Boolean, Character
  • Access Modifiers: public, private, protected, default
  • Non-access Modifiers: static, final, abstract, synchronized
  • Keywords: this, super, static, final, abstract, interface
  • String Pool: Understanding String immutability and memory management
  • equals() vs ==: Difference between reference and value comparison
  • ArrayList vs Array: Dynamic vs fixed size
  • Checked vs Unchecked Exceptions: IOException vs NullPointerException

Preparation Tips

  • Practice writing complete Java programs with proper class structure
  • Understand the difference between compilation and runtime errors
  • Master the four pillars of OOP: Abstraction, Encapsulation, Inheritance, Polymorphism
  • Practice tracing code execution, especially with inheritance and polymorphism
  • Review Java API documentation for commonly used classes
  • Work on debugging exercises to identify common errors
  • Practice collection framework operations and their time complexities
  • Understand memory management: Stack vs Heap
  • Review exception hierarchy and how to handle different exception types

Common Mistakes to Avoid

  • Forgetting semicolons at the end of statements
  • Incorrect capitalization in keywords and class names
  • Missing curly braces in control structures
  • Comparing strings with == instead of .equals()
  • Not initializing variables before use
  • Incorrect array indexing (ArrayIndexOutOfBoundsException)
  • Missing import statements for classes
  • Forgetting to call super() in constructors when needed
